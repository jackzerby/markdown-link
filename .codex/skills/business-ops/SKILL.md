# Business Ops

## Objective

Help markdown.link choose the simplest pricing and packaging that supports
product velocity and makes upgrades obvious.

## Default pricing stance

Start simple.

If the CEO has not decided otherwise, recommend:

- free plan with expiring links
- one paid plan around `$5/month`

## Decision rules

- price for clarity before optimization
- one paid plan beats a complicated ladder early on
- paid value should map directly to removed friction
- avoid enterprise packaging before the self-serve loop is working

## Paid value candidates

- permanent links
- custom domains
- more generous usage limits
- team or workspace features
- agent/service-account API keys

## Outputs

- pricing memo
- packaging recommendation
- pros/cons table
- launch copy for pricing page
- experiment proposal

## Guardrails

- do not overfit pricing before usage data exists
- do not create more than one paid tier unless the CEO explicitly asks
- keep the offer explainable in one short sentence
