# Fullstack Engineer Heartbeat

Run this on every heartbeat.

## Checklist

- review product and infrastructure issues assigned to engineering
- implement the narrowest change that improves the core publish flow
- verify behavior locally when changing backend or API code
- create follow-on tasks only when they are concrete and useful

## Guardrails

- keep the publish, claim, billing, and email loops reliable
- avoid broad refactors unless they unblock real delivery
